# 第四章文本检测实践要求整理

本文档整理自 `PPT/PDF/第四章 文本检测实践.pdf`，作为后续实现、报告和答辩的对照清单。

## 选题

第三部分题目为：基于字符相似性网络的对抗垃圾文本检测模型实践示范。

## 指定数据集

课程第三部分明确使用 AST 数据集，即 adversarial spam text 数据集。

- 数据来源：课件描述为来自 GitHub。
- 数据规模：共 16008 条文本。
- 类别分布：正常文本 5000 条，垃圾文本 11008 条。
- 数据特点：主要包含字形相似、字音相似等变体形式的垃圾文本。
- 标签含义：`1` 表示垃圾文本，`0` 表示正常文本。

当前项目数据策略：

- 优先：拿到 AST 原始文件后，作为正式主数据集。
- 备选：使用公开中文垃圾短信带标签数据集跑工程 baseline。
- 补充：由普通垃圾文本生成字形/字音扰动样本，用于模拟 AST 中的对抗变体，并服务于展示。

## 算法流程

课程要求的核心算法流程：

1. 数据集介绍
2. 字形编码和字音编码
3. 构建字符相似性网络
4. 利用字符相似性网络进行字符嵌入学习
5. 生成句子嵌入
6. 构建分类模型

## 工程流程

课程前半部分给出的通用文本检测流程：

1. 数据收集
2. 数据预处理：去除 HTML/XML、特殊字符、标点、处理空文本等
3. 标记化：中文可使用 jieba 分词，也可使用字符级切分
4. 停用词处理
5. 特征提取：BOW、TF-IDF、N-gram、Word2Vec 等
6. 分类模型：朴素贝叶斯、SVM、逻辑回归、深度学习模型等
7. 模型评估与调优

## 课程中的简单样例

课件中的简单样例包含：

- `hit_stopwords.txt`：停用词表
- `train.txt`：训练集，正样本标签为 1，负样本标签为 0
- `test.txt`：测试集

样例流程包含读取数据、文本预处理、Word2Vec 特征提取、SVM 分类、混淆矩阵评估、过采样、网格搜索调参等。

## 系统评测与验证

课程第三部分要求做以下分析：

1. 模型对比分析：对比使用字符相似性网络前后的分类表现。
2. 超参分析：重点分析汉字相似度阈值对模型表现的影响。
3. 样例分析：展示模型识别出的对抗垃圾文本，如“薇信、胃星、维信、卫星”等。
4. 与基准算法对比：课件列出的基准包括 GAS、Word2Vec-w+LR、Word2Vec-c+LR、Word2Vec-c+GBDT、Doc2Vec-c+GBDT，并与 ASD-CSN 对比。

## 报告可写重点

- 为什么普通词级模型难以识别“微信”的字形/字音变体。
- 字符相似性网络如何把变体字与原始敏感词联系起来。
- 对抗样本上的召回率提升，比普通准确率更能说明模型价值。
- 阈值变化曲线、混淆矩阵、典型样例表格适合用于结果展示。
