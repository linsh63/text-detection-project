# Bad Case 分析

本文件分析 v1 在 AST 测试集上的错误样本。v1 的测试集误报 `11` 条，漏检 `18` 条。

## 漏检 FN 特点

漏检样本是真实垃圾文本，但模型预测为正常文本。主要特点如下：

1. 垃圾意图被长正常句子包裹  
   部分样本使用祝福、鸡汤、通知等正常文本做主体，只在末尾加入网址、退订、联系方式等垃圾线索。

2. 关键线索被分隔符打散  
   例如使用 `*`、空格、括号等符号拆开联系方式或业务描述，清洗后上下文变得稀疏。

3. 垃圾文本很像普通通知  
   某些营销短信、系统通知、活动提醒在语言风格上接近正常通知，单靠字符 n-gram 难以区分真实意图。

4. 短链接和数字占比较高  
   一些样本主要由网址、编号、电话和短句构成，语义信息少，模型容易给出较低垃圾分数。

## 漏检例子

| 类型 | 样本文本片段 | 说明 |
|---|---|---|
| 正常话术包裹 | `罗马不是一天建成的...新春佳节...回r退订` | 主体像祝福短信，垃圾线索只在末尾 |
| 网址 + 励志话术 | `不要向这个世界认输...3368tf.c0m/9gcv...` | 垃圾入口隐藏在正常句子中 |
| 联系方式打散 | `提*供他的号*码...TEL１５７...` | 分隔符和全角数字削弱模式 |
| 招聘刷单 | `刷单加qq...多劳多得...` | 训练中相似表达不足时会漏检 |

## 误报 FP 特点

误报样本是真实正常文本，但模型预测为垃圾文本。主要特点如下：

1. 正常文本含价格、优惠、票价等商业词  
   这些词和垃圾短信中的促销表达相似，容易触发垃圾类别特征。

2. 正常文本含链接  
   普通博客、网页链接也可能被模型误认为垃圾入口。

3. 情绪化短句  
   很短的情绪表达缺少上下文，字符特征容易被少量高权重 n-gram 影响。

4. 重复字符过多  
   长串重复字符不常见，模型可能把异常文本形态和垃圾类别关联起来。

## 误报例子

| 类型 | 样本文本片段 | 说明 |
|---|---|---|
| 商业正常信息 | `优惠场票价 成人40元 学生30元...` | 价格和优惠表达接近广告 |
| 普通链接 | `http://blog.yam.com/...` | 链接本身不是垃圾，但模型容易提高风险 |
| 情绪短句 | `你赶快去看心理医生！` | 文本过短，缺少上下文 |
| 重复字符 | `哈哈哈哈哈哈...` | 异常重复形态导致误判 |

## 改进方向

当前我们决定只保留 v1，因此不引入人工规则。后续可考虑的改进应保持“模型自动学习”的原则：

- 增加更多真实正常通知、票价、链接类样本，减少误报。
- 增加真实垃圾中长话术、短链接、联系方式变体样本，减少漏检。
- 对训练集做更规范的数据清洗和去重，避免数据噪声影响字符 n-gram 权重。
- 使用交叉验证确认 v1 的稳定性，而不是只依赖一次 70/30 划分。
