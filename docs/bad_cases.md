# Bad Case 分析

本文件分析 AST 测试集上的错误样本。TF-IDF+SVM 的误报为 `13` 条、漏检为 `16` 条；课件版 Rule-free CSN+LR 的误报为 `50` 条、漏检为 `87` 条。

## 漏检 FN 特点

漏检样本是真实垃圾文本，但模型预测为正常文本。主要特点如下：

1. **垃圾意图被长正常句子包裹**  
   一些样本主体像新闻、通知、祝福或普通叙述，只在局部出现网址、联系方式或退订信息。

2. **关键线索被分隔符打散**  
   例如使用 `*`、空格、括号、全角数字等方式拆开联系方式或业务描述。TF-IDF+SVM 还能利用局部 n-gram，CSN 句向量池化后更容易被稀释。

3. **短链接和数字占比较高**  
   课件版 CSN 主要依赖字符嵌入和句子嵌入，不直接记忆高维 URL/数字串模板，因此这类样本更容易漏检。

4. **营销通知接近正常通知**  
   流量提醒、会员权益、影视限免、活动提醒等文本风格接近正常服务通知，模型需要更细粒度的上下文才能区分。

## 漏检例子

| 类型 | 样本文本片段 | 说明 |
|---|---|---|
| 新闻话术包裹 | `广州天河北：z股未能迎来七月开门红...` | 长文本主体像财经新闻，垃圾标签线索不集中 |
| 普通通知风格 | `香港寬頻:你的本地數據用量已達至每月用量上限...` | 接近真实运营商通知 |
| 联系方式打散 | `只要*有他的*号、码...请拨159...` | 分隔符削弱局部模式 |
| 短链接 | `总要热爱点什么... z9r.me/VKL...` | 垃圾入口被短链接承载 |

## 误报 FP 特点

误报样本是真实正常文本，但模型预测为垃圾文本。主要特点如下：

1. **论坛格式复杂**  
   含发信人、标题、版面、转发符号、ID、日期等结构化片段，文本形态接近垃圾短信中的异常格式。

2. **正常文本含电话、数字或名单**  
   报失信息、名单统计、联系方式等正常内容也会出现大量数字和姓名，容易被误判。

3. **普通链接或网络地址**  
   正常网页地址和垃圾入口在字符层面很像。

4. **长文本主题稀释**  
   CSN 句向量把整段文本压缩成一个低维向量，部分正常长文本会被异常片段拉偏。

## 误报例子

| 类型 | 样本文本片段 | 说明 |
|---|---|---|
| 论坛转发格式 | `发信人... 标题... 本日十大衷心祝福...` | 结构化符号和 ID 较多 |
| 名单统计 | `人数 amw 1 gg...` | 大量短 token 和数字 |
| 校园信息 | `报失：徐曼 135... IC卡...` | 正常联系电话被误认为风险线索 |
| 医疗问答 | `深度近视是否应剖腹产...` | 长文本中局部表达被放大 |

## 两个模型的取舍

| 方法 | FP | FN | 主要特点 |
|---|---:|---:|---|
| TF-IDF+SVM | 13 | 16 | 稀疏字符 n-gram 很强，误报和漏检都少 |
| Rule-free CSN+LR | 50 | 87 | 更符合课件 CSN 流程，但低维句向量会损失 URL、数字串和模板细节 |

因此，最终展示时建议把 TF-IDF+SVM 作为强 baseline，把 Rule-free CSN+LR 作为课程指定思路的实现版本。报告中需要说明：CSN 版本不是为了刷分，而是为了展示字符相似性网络、字符嵌入和句子嵌入的完整流程。

## 改进方向

- 增加更真实的正常通知、论坛格式、电话名单类样本，减少误报。
- 增加短链接、联系方式变体、长文本包裹广告类样本，减少漏检。
- 对 CSN 的相似度阈值、邻居数量、句向量池化方式做超参分析。
- 使用多次随机划分或去重后评测，确认指标是否稳定。
